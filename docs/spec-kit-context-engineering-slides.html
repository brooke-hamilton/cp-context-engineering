Bootstrap Process

Steps to Initialize a New Repository

  1. Clone the context engineering repo or use as template
  2. Run bootstrap script with project parameters
  3. Customize the constitution for your specific project
  4. Select applicable instruction files based on tech stack
  5. Configure agents and skills for your use cases
# Example bootstrap command
./scripts/bootstrap.sh \
  --name "my-new-project" \
  --tech "typescript,react,node" \
  --template "web-app"

Instruction File Example

Organization-level instruction file for shell scripting:

<instructions>
You are writing shell scripts for the Radius platform.

<rules>
- Use bash with strict mode: set -euo pipefail
- Always quote variables: "${variable}"
- Check command existence before use
- Include meaningful error messages
- Use functions for reusable logic
</rules>

<conventions>
- Script naming: lowercase-with-hyphens.sh
- Environment variables: UPPERCASE_WITH_UNDERSCORES
- Exit codes: 0 for success, 1 for general errors
</conventions>
</instructions>
